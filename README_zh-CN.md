# AI 辅助顽固 Bug 诊断器（三轮法则）

## 概述

本项目提供了一套系统化的方法论和工具，旨在帮助 AI 助手诊断和解决那些顽固的软件 Bug，尤其是在传统调试方法多次尝试后仍然无效的情况下。受“三轮法则”启发，本框架指导 AI 助手超越“打地鼠式”的修补，转向结构化的分析、回滚和最小可验证修复（MVF）方法。

## 动机

在 AI 辅助的软件开发中，AI 模型根据即时错误信息尝试快速修复是很常见的。尽管这通常很有效，但也可能导致“打地鼠”式的困境——修复一个问题又引入另一个，或者同一个 Bug 反复出现。“三轮法则”旨在当 Bug 变得顽固时，通过强制执行一套严谨的诊断流程来打破这种循环。

## 核心方法论：三轮法则

当一个 Bug 在连续 2-3 轮修复尝试后仍然存在，或者开发者感到“陷入循环”时，AI 助手必须切换到这种系统化的诊断模式。

### 1. 触发机制

-   **自动触发**：同一错误信息或测试失败在连续 2-3 轮修复尝试后未改变或反复出现。
-   **手动触发**：用户明确输入关键词，例如 `timeout`、`still error`、`3 rounds`、`顽固诊断` 或 `好像绕回去了`。

### 2. 核心执行流程

#### 步骤 1：状态快照与回滚确认

在进行任何新的尝试之前，必须建立一个清晰的基线：

1.  **获取完整错误**：向用户请求最新的完整堆栈跟踪（Stack Trace）。
2.  **识别回滚点**：找到最后一个已知能正常工作的版本（例如，Git commit hash、上一次成功的对话轮次或备份文件）。
3.  **清理环境**：如果当前代码因多次尝试而变得混乱，建议回滚到已知良好状态以进行重新分析。

#### 步骤 2：诊断决策树（错误原因分类）

按顺序系统地检查以下四个维度。**严禁跳过维度。**

| 维度 | 诊断标准 | 典型表现 | 行动方向 |
| :-------- | :------------------ | :--------------------- | :--------------- |
| **A. 局部逻辑错误** | 错误指向特定、孤立的代码逻辑 | 空指针、索引越界、类型不匹配、除零错误 | 重写特定函数/模块的单元测试，隔离验证 |
| **B. 上下文/状态污染** | 错误位置飘移，修复 A 导致 B 出现新问题 | 死锁、竞态条件、全局变量冲突、缓存失效 | 回滚，然后侧重于改变变量作用域/数据流，而非修复直接报错点 |
| **C. 上游数据/外部依赖** | 错误仅在特定输入或环境下出现 | API 响应格式变化、文件编码问题、时间/时区差异 | 向用户请求最小输入样本，模拟外部调用进行测试 |
| **D. AI 模型盲区** | 涉及复杂并发、反射、底层系统调用 | 异步回调顺序、泛型擦除、内存泄漏 | 停止直接代码修改，生成详细诊断日志，请求人工审查 |

#### 步骤 3：最小可验证修复（MVF）

输出必须且只能包含：

1.  **单一核心改动**：严禁一次性进行多个不相关的修改。
2.  **验证脚本/命令**：提供具体的测试命令（例如 `pytest`、`curl`）和预期输出。
3.  **回滚计划**：明确说明如果修复失败如何撤销更改（例如 `git revert <commit>` 或注释掉更改）。

#### 步骤 4：内部自检清单（AI 内部执行，不输出给用户）

在回复用户之前，AI 必须自问：

-   这个解决方案是否需要我尚未获取的新信息？（如果是，则首先请求信息。）
-   如果用户执行我的解决方案后仍然失败，最可能的次要原因是什么？（告知用户此预期。）
-   是否存在更简单的替代方案（例如，禁用某个功能、降级依赖、使用不同的算法）？（如果存在，优先推荐。）

### 3. 辅助工具

-   `src/bug_cracker.py`：一个 Python 脚本，用于辅助分析错误日志并根据关键词建议诊断维度。

### 4. AI 助手的最佳实践

-   **避免含糊不清**：不要说“或许可以尝试...”，而应说“基于维度 B 的诊断，我建议进行以下修改...”。
-   **强制测试**：每个提议的修复都必须附带一个验证步骤。
-   **尊重回滚**：如果连续两次 MVF 尝试失败，强烈建议回滚到步骤 1 的快照点。

## 与 AI 助手的集成

本方法论可以集成到各种 AI 助手平台中：

-   **系统提示（System Prompt）**：将核心执行流程嵌入到 AI 的系统提示中，作为通用指导。
-   **自定义工具/技能（Custom Tool/Skill）**：作为自定义工具或技能实现（例如 Manus Skill、Cursor Skill），可以显式调用或自动触发。
-   **IDE 扩展**：开发为 IDE（例如 VS Code Copilot、Cursor）的扩展，提供上下文感知的建议。

## 安装与使用

### 用于 AI 系统提示集成

请参考 `docs/system_prompt_template.md` 获取即用型系统提示模板。

### 用于 IDE（例如 Cursor、带有 Copilot 的 VS Code）

请参考 `rules/ide_rules.md` 获取可添加到 `.cursorrules` 或类似 AI 配置文件中的配置片段。

### 用于 `bug_cracker.py`

```bash
python src/bug_cracker.py <错误日志文件路径>
```
